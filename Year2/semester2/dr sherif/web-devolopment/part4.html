0. Introduction to Security

How to secure this application using User Authentication (Login/Signup) so users can only see their own tasks?

To secure a web application, we implement Authentication (who are you?) and Authorization (what are you allowed to do?). The industry standard for modern web apps is using JWT (JSON Web Tokens).

1. How JWT Authentication Works

Instead of the server "remembering" you (which uses a lot of memory), the server gives you a "digital pass" (the Token) after you log in. You show this pass every time you request data.

  • Login: User sends credentials (email/password).
  • Verification: Server checks the database. If correct, it signs a JWT using a Secret Key.
  • Storage: The Client (React) saves this token in localStorage or a Cookie.
  • Access: For every future request, React sends the token in the Header.
  • Validation: The Server checks the signature. If valid, it sends the data.

4. Security Best Practices
  • Password Hashing: Never store plain-text passwords. Use a library like bcrypt to scramble them before saving to the database.
  • Environment Variables: Store your TOKEN_SECRET and Database URLs in a .env file, never in your actual code.
  • HTTPS: Always use encryption so hackers can't "sniff" the tokens in transit.
